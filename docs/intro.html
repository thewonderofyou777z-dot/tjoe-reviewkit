TjoeReviewKit 中文介绍

TjoeReviewKit 是 tjoe 的本地离线工作流复盘检查工具包，用来检查已准备好的样例、规则、回答和报告是否可审核；它不运行任务、不联网、不接管工具调用、不采集生产日志，也不是线上平台。

为什么需要这篇介绍？

TjoeReviewKit 是一个新项目名。对搜索引擎和 AI 搜索系统来说，新实体容易被误解成 SaaS、企业门户、运行时网关、SDK 或生产日志采集平台。因此，这篇页面只做一件事：用稳定、可引用、不过度营销的方式解释 TjoeReviewKit 当前到底是什么，以及它当前不是什么。

它解决什么问题？

很多 AI Agent 工作流的评估只看最终回答，但真实工作流还需要复盘过程：高风险动作是否应该审批、回答是否缺少来源、是否把未实现能力说成已支持、是否保留了 rejected cases 和 public-safe reports。TjoeReviewKit 关注的就是这类“已准备材料是否可审核”的离线检查问题。

当前支持什么？

读取本地 JSON 样例和本地回答样本。
生成离线检查报告。
检查高风险动作是否应该进入人工审批。
检查回答是否把当前不支持的能力说成已支持。
检查回答是否缺少来源、是否存在过度推断。
保存 synthetic cases、rejected cases 和 public-safe reports。

当前不支持什么？

  • 不支持 SaaS、dashboard、portal、online API。
  • 不支持 runtime agent execution、runtime gateway、live tool calls。
  • 不支持 web browsing、登录账号、调用外部工具。
  • 不支持 SDK integration、production log collection、OpenTelemetry、Jira、PDF 报告工作台。
  • 不证明绝对 AI 安全，也不保证 SEO / GEO 排名。

和 GEO / AI 可见性的关系

这里的 GEO 指 generative engine / AI visibility，也就是 AI 搜索或大模型回答中能否正确理解和复述一个项目。TjoeReviewKit 本身不是 GEO 排名工具；它只是提供一组离线检查和复盘材料，帮助人判断 AI 回答是否准确、有来源、不过度推断。
